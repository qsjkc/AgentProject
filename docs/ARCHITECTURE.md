```
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌───────────────┐ │
│  │对话服务 │ │ RAG服务 │ │工具服务 │ │ 用户认证      │ │
│  └─────────┘ └─────────┘ └─────────┘ └───────────────┘ │

```

## 模块说明

### 前端模块

| 模块 | 职责 |
|------|------|
| pages/ | 页面组件（Chat、Knowledge、Login） |
| components/ | 可复用UI组件 |
| hooks/ | 自定义React Hooks |
| stores/ | Zustand状态管理 |
| services/ | API调用封装 |
| types/ | TypeScript类型定义 |

### 后端模块

| 模块 | 职责 |
|------|------|
| api/ | API路由定义 |
| core/ | 核心配置（配置、安全、日志） |
| services/ | 业务逻辑层 |
| models/ | SQLAlchemy数据模型 |
| schemas/ | Pydantic请求/响应模型 |

## 数据流

### 对话流程

```
用户输入 → 前端组件 → API请求 → 后端路由 → LLM服务 → 流式响应 → 前端渲染
                ↓                              ↓
            状态管理                      数据库存储
```

### RAG流程

```
文档上传 → 文本分块 → 向量化 → Chroma存储
                                    ↓
用户提问 → 向量检索 → 上下文组装 → LLM调用 → 返回答案
```

## 技术决策

### 为什么选择SQLite？

- **轻量级**: 适合2核2G服务器
- **零配置**: 无需独立进程
- **性能足够**: 对于中小规模应用完全够用
- **易于迁移**: 后续可平滑迁移到PostgreSQL

### 为什么选择Chroma？

- **嵌入式**: 无需独立服务，节省内存
- **易用性**: API简单，与LangChain集成良好
- **持久化**: 支持本地存储

### 为什么选择DeepSeek？

- **免费额度**: 500万tokens/月
- **OpenAI兼容**: API接口标准
- **性能优秀**: 中文能力强

## 扩展性设计

### 水平扩展

- 使用Docker容器化部署
- 可通过增加容器实例扩展
- 需要配置负载均衡

### 功能扩展

- **多模型支持**: 通过LLMProvider接口扩展
- **工具扩展**: 通过ToolRegistry注册新工具
- **存储扩展**: 可替换SQLite为PostgreSQL
